# 🔐 SecureVault — Менеджер паролей с AMPG v1

**Безопасный менеджер паролей для Android с авторским алгоритмом мнемонической генерации.**

Проект разработан в рамках магистерской диссертации.

---

## 🎯 Цель проекта

Создать безопасное приложение для локального хранения паролей с двумя режимами генерации:
- **Обычный** — криптостойкие случайные пароли
- **Мнемонический (AMPG v1)** — запоминающиеся пароли по пользовательской фразе

---

## ✨ Основные функции

### 🔑 Генерация паролей
- **Обычный генератор**: SecureRandom, без повторяющихся символов
- **Мнемонический генератор AMPG v1**: детерминированная генерация по фразе
- **5 вариантов** в каждом наборе с разными стратегиями
- **Кнопка "Ещё варианты"** для перегенерации

### 🔄 Ротация паролей
- Индивидуальная ротация с напоминаниями
- Массовая ротация с предпросмотром
- **Перемешивание паролей между сервисами** внутри профиля
- Автоматическое обновление `nextRotationDate`

### 📜 История паролей
- Зашифрованное хранение старых паролей
- Просмотр только после подтверждения мастер-пароля
- Типы операций: random / mnemonic / manual / shuffled / swapped
- Связанные сервисы и записи

### 🛡️ Безопасность
- AES-256-GCM через Android Keystore
- PBKDF2 для мастер-пароля (100,000 итераций)
- HMAC-SHA256 для fingerprint паролей
- Constant-time compare
- BruteForceGuard с блокировкой
- Очистка charArray после использования

### 📤 Импорт/Экспорт
- CSV с предупреждениями о безопасности
- Корректная обработка quoted полей
- Генерация новых UUID при импорте
- Поддержка разных профилей

### 🔍 Аудит безопасности
- Поиск слабых паролей
- Проверка просроченной ротации
- Обнаружение повторного использования паролей
- Проверка 60% уникальности

### 📱 QR-коды
- Генерация защищённых QR для записей
- QR НЕ содержит пароль
- Привязка к устройству и профилю
- Срок действия 24 часа

## 🧠 AMPG v1 — Авторский алгоритм

**AMPG** = Adaptive Mnemonic Password Generation  
**АМГП** = Адаптивная Мнемоническая Генерация Паролей

### Входные данные
- `phrase` — пользовательская фраза ("моя кошка любит рыбу")
- `serviceName` — название сервиса ("Gmail")
- `rotationDate` — дата ротации (июнь 2026)
- `targetLength` — целевая длина
- `includeLeet` — leet-замены (a→@, o→0, e→3, i→1, s→$)
- `includeServiceCode` — код сервиса
- `includeRotationCode` — код ротации
- `variantOffset` — смещение варианта

### Стратегии генерации
1. **BlockStrategy** (5): CONSONANTS, FIRST_LAST, FIRST_VOWELS, SYLLABLE_LIKE, INITIALS_ONLY
2. **OrderStrategy** (4): ORIGINAL, REVERSED, ALTERNATING, LONG_FIRST
3. **CasingStrategy** (4): CAPITALIZE_BLOCKS, ALTERNATING_CASE, SERVICE_UPPER, FIRST_LAST_UPPER
4. **LeetStrategy** (4): NONE, SOFT, FULL, VOWEL_ONLY
5. **TokenOrderStrategy** (4): PHRASE_SERVICE_ROTATION, SERVICE_PHRASE_ROTATION, PHRASE_ROTATION_SERVICE, PHRASE_ONLY
6. **ServiceCodeStyle** (5): FIRST2_LENGTH, FIRST3, FIRST_LAST, CONSONANTS, NO_SERVICE
7. **RotationCodeStyle** (5): MMYY, YYMM, MM_YY, QYY, NO_ROTATION
8. **SeparatorStrategy** (4): DASH, DOT, NONE, MAJOR_ONLY

### Пример
**Вход:** "моя кошка любит рыбу" + Gmail + июнь 2026  
**Результат:** `Mya-Kshk-Lbt-Ryb-Gm5-0626`  
**Подсказка:** "моя кошка любит рыбу + сервис + ротация [CONSONANTS, ORIGINAL]"

### Важно
**Это НЕ криптография.** Алгоритм предназначен для создания **запоминаемых** паролей, а не для защиты данных. Для реальной безопасности используйте обычный генератор.

## 🔒 Безопасность

### Мастер-пароль
- Минимум 12 символов
- PBKDF2WithHmacSHA256 (100,000 итераций)
- Соль 16 байт
- Constant-time compare
- BruteForceGuard: 5 попыток → 30 сек, 7 → 1 мин, 10 → сброс

### Шифрование паролей
- AES-256-GCM
- Ключ в Android Keystore
- Уникальный IV для каждого пароля
- Base64.NO_WRAP

### Fingerprint паролей
- HMAC-SHA256 с ключом из Keystore
- Защита от утечки базы
- Совместимость со старыми SHA-256

### Валидация новых паролей
1. ✅ Нет повторяющихся символов
2. ✅ Не использовался ранее для этого сервиса
3. ✅ Отличается от предыдущего минимум на 60%
4. ✅ Проверка по fingerprint, не по ciphertext

### История
- Старые пароли зашифрованы
- Просмотр только после подтверждения
- Автоматическое скрытие при закрытии

## ⚠️ Ограничения

### Импорт/Экспорт
- CSV **не предназначен** для безопасной передачи между устройствами
- Android Keystore привязан к устройству
- Для межплатформенного переноса нужен защищённый экспорт (TODO)

### QR-коды
- Работают только на том же устройстве
- Требуют мастер-пароль для просмотра
- Срок действия 24 часа

### Мнемонические пароли
- Менее криптостойкие, чем случайные
- Предназначены для удобства, а не максимальной безопасности
