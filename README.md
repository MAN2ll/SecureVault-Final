
#  SecureVault — Менеджер паролей с AMPG v1

**Безопасный менеджер паролей для Android с авторским алгоритмом мнемонической генерации.**

Проект разработан в рамках магистерской диссертации.

**Репозиторий:** https://github.com/MAN2ll/SecureVault-Final

---

##  Цель проекта

Создать безопасное Android-приложение для локального хранения паролей с двумя режимами генерации:

1. **Обычный генератор** — криптостойкие случайные пароли без повторяющихся символов
2. **Мнемонический генератор AMPG v1** — запоминающиеся пароли по пользовательской фразе

Приложение решает проблему баланса между **безопасностью** (сложные пароли) и **удобством** (возможность запомнить пароль по подсказке).

---

##  Основные функции

###  Генерация паролей
- **Обычный генератор**: SecureRandom, без повторяющихся символов (A и a считаются повтором)
- **Мнемонический генератор AMPG v1**: детерминированная генерация по фразе
- **5 вариантов** в каждом наборе с разными стратегиями
- **Кнопка "Ещё варианты"** для перегенерации с номером набора

###  Ротация паролей
- Индивидуальная ротация с напоминаниями (3, 6, 12 месяцев)
- Массовая ротация с предпросмотром
- **Перемешивание паролей между сервисами** внутри профиля
- Автоматическое обновление `nextRotationDate`

###  История паролей
- Зашифрованное хранение старых паролей (AES-256-GCM)
- Просмотр только после подтверждения мастер-пароля
- Типы операций: `random`, `mnemonic`, `manual`, `shuffled`, `swapped`
- Связанные сервисы и записи

###  Безопасность
- AES-256-GCM через Android Keystore для шифрования паролей
- PBKDF2WithHmacSHA256 (100,000 итераций) для мастер-пароля
- HMAC-SHA256 для fingerprint паролей (ключ в Android Keystore)
- Constant-time compare для защиты от timing attacks
- BruteForceGuard: блокировка после неудачных попыток
- Очистка charArray после использования

###  QR-коды
- Генерация защищённых QR для записей
- QR **НЕ содержит пароль** — только защищённый токен
- Привязка к устройству и профилю
- Действует до смены пароля (проверка `lastChanged`)
- Требует мастер-пароль для просмотра
- Сканирование камерой внутри приложения (CameraX + ML Kit)

###  Импорт/Экспорт
- CSV v2 с новыми полями (fingerprint, mnemonicPhraseHint)
- Обратная совместимость со старым форматом
- Корректная обработка quoted полей
- Генерация новых UUID при импорте
- Поддержка разных профилей

###  Аудит безопасности
- Поверхностный аудит по метаданным
- Глубокий аудит после подтверждения мастер-пароля
- Проверка длины, повторов, структуры

###  Мнемоническая генерация AMPG v1
- 8 стратегий построения пароля
- 5 вариантов в каждом наборе
- Кнопка "Ещё варианты" с номером набора
- Сброс номера при изменении параметров

##  AMPG v1 — Авторский алгоритм

**AMPG** = Adaptive Mnemonic Password Generation  
**АМГП** = Адаптивная Мнемоническая Генерация Паролей

###  Важно понимать

**AMPG — это генератор, а НЕ криптография.** Алгоритм предназначен для создания **запоминаемых** паролей, а не для защиты данных. Для реальной безопасности используйте обычный генератор.

### Входные данные
- `phrase` — пользовательская фраза ("моя кошка любит рыбу")
- `serviceName` — название сервиса ("Gmail")
- `rotationDate` — дата ротации (июнь 2026)
- `targetLength` — целевая длина
- `includeLeet` — leet-замены (a→@, o→0, e→3, i→1, s→$)
- `includeServiceCode` — код сервиса
- `includeRotationCode` — код ротации
- `variantOffset` — смещение варианта

### Стратегии генерации

| Стратегия | Варианты | Описание |
|-----------|----------|----------|
| **BlockStrategy** | 5 | CONSONANTS, FIRST_LAST, FIRST_VOWELS, SYLLABLE_LIKE, INITIALS_ONLY |
| **OrderStrategy** | 4 | ORIGINAL, REVERSED, ALTERNATING, LONG_FIRST |
| **CasingStrategy** | 4 | CAPITALIZE_BLOCKS, ALTERNATING_CASE, SERVICE_UPPER, FIRST_LAST_UPPER |
| **LeetStrategy** | 4 | NONE, SOFT, FULL, VOWEL_ONLY |
| **TokenOrderStrategy** | 4 | PHRASE_SERVICE_ROTATION, SERVICE_PHRASE_ROTATION, PHRASE_ROTATION_SERVICE, PHRASE_ONLY |
| **ServiceCodeStyle** | 5 | FIRST2_LENGTH, FIRST3, FIRST_LAST, CONSONANTS, NO_SERVICE |
| **RotationCodeStyle** | 5 | MMYY, YYMM, MM_YY, QYY, NO_ROTATION |
| **SeparatorStrategy** | 4 | DASH, DOT, NONE, MAJOR_ONLY |

### Пример

**Вход:**
- Фраза: "моя кошка любит рыбу"
- Сервис: Gmail
- Ротация: июнь 2026
- variantOffset = 0

**Результат:**
```
Mya-Kshk-Lbt-Ryb-Gm5-0626
```

**Подсказка:**
```
моя кошка любит рыбу + сервис + ротация [CONSONANTS, ORIGINAL]
```

### Детерминированность

**Одинаковые входные данные = одинаковый пароль.** Это позволяет:
- Восстанавливать пароль по подсказке
- Не хранить пароль в открытом виде
- Использовать QR-коды для быстрого доступа

##  Безопасность

### Мастер-пароль
- **Минимум 12 символов**
- PBKDF2WithHmacSHA256 (100,000 итераций)
- Соль 16 байт (случайная)
- Constant-time compare
- BruteForceGuard:
  - 5 неудачных попыток → блокировка 30 сек
  - 7 неудачных попыток → блокировка 1 мин
  - 10 неудачных попыток → сброс доступа

### Шифрование паролей
- AES-256-GCM
- Ключ в Android Keystore
- Уникальный IV для каждого пароля
- Base64.NO_WRAP

### Fingerprint паролей
- HMAC-SHA256 с ключом из Android Keystore
- Защита от утечки базы (fingerprint нельзя использовать для восстановления пароля)
- Совместимость со старыми SHA-256 fingerprint (fallback)

### Валидация новых паролей
1.  Нет повторяющихся символов (без учёта регистра)
2.  Не использовался ранее для этого сервиса (проверка по fingerprint)
3.  Отличается от предыдущего минимум на 60%
4.  Проверка работает по fingerprint, не по ciphertext

### История
- Старые пароли зашифрованы через AES-256-GCM
- Просмотр только после подтверждения мастер-пароля
- Автоматическое скрытие при закрытии диалога

### Профильный PIN
- **Короткий локальный PIN** для разделения профилей
- **НЕ замена мастер-пароля**
- Мастер-пароль защищает всё хранилище
- Профильный PIN нужен только для локальной навигации

### QR-коды
- QR **НЕ содержит пароль** — только защищённый токен
- Токен содержит: `entryId`, `profileIdHash`, `deviceBindingHash`, `nonce`, `lastChanged`
- Привязка к конкретному профилю и устройству
- Действует до смены пароля (проверка `lastChanged`)
- Требует мастер-пароль для просмотра
- Обычная камера телефона **НЕ покажет пароль**

##  Ограничения

### Импорт/Экспорт
- CSV **не предназначен** для безопасной передачи между устройствами
- CSV содержит зашифрованные пароли + метаданные (подсказки, fingerprint)
- Android Keystore привязан к устройству — пароли нельзя расшифровать на другом устройстве
- Для межплатформенного переноса нужен защищённый экспорт с отдельным паролем (TODO)

### QR-коды
- Работают только на том же устройстве
- Требуют мастер-пароль для просмотра
- Действуют до смены пароля

### Мнемонические пароли
- Менее криптостойкие, чем случайные
- Предназначены для удобства, а не максимальной безопасности

### BruteForceGuard
- После 10 неудачных попыток происходит **сброс доступа** (очистка `auth_prefs`)
- **База паролей НЕ удаляется** — это сознательное решение для защиты данных пользователя
- Если пользователь забудет мастер-пароль, данные останутся зашифрованными

##  Сценарии ручной проверки

### 1. Первый запуск
-  Создание мастер-пароля (минимум 12 символов)
-  Проверка совпадения пароля и подтверждения
-  Переход к списку профилей

### 2. Профили
-  Создание профиля с коротким PIN
-  Вход в профиль
-  Выход из профиля → возврат к списку

### 3. Обычный генератор
-  Создать запись → нажать 
- Сгенерированный пароль **не содержит повторяющихся символов**
-  Кнопка "Ещё раз" создаёт новый пароль
-  Кнопки "Копировать" и "Ещё раз" расположены вертикально

### 4. Мнемонический генератор (AMPG v1)
- Нажать  → ввести фразу "моя кошка любит рыбу" + сервис "Gmail"
- Появляются **5 вариантов** с разными названиями стратегий
- Кнопка "Ещё варианты" → **все 5 вариантов меняются**
- Номер набора: "Текущий набор: №1" → "№2" → "№3"
- Выключить "Leet" → все варианты без замен
- Выключить "Код сервиса" → все варианты без кода сервиса
- Изменить фразу → номер набора сбрасывается на №1

### 5. Ротация
- Включить ротацию в записи → запись появляется в "Ротация паролей"
- Кнопка "Заменить" → диалог с 3 режимами (Случайный/Мнемонический/Вручную)
- После замены → `nextRotationDate` обновляется
- В поле фразы при мнемонической ротации → **только короткая фраза** (не вся подсказка)

### 6. Перемешивание паролей
- Выбрать 3+ сервиса в одном профиле
- "Перемешать между сервисами" → построить схему
- Видно: `GitHub → Gmail → VK → Work PC → GitHub`
- "Применить" → все пароли перемешаны
- Каждый сервис получил **чужой** пароль (не свой)

### 7. История
- Открыть запись → "Показать историю"
-  Без мастер-пароля → пароли скрыты ("Старый пароль скрыт")
- Подтвердить мастер-пароль → видны старые пароли
- Кнопка копирования работает
- Закрыть диалог → при повторном открытии всё снова скрыто

### 8. Смена мастер-пароля
- Настройки → "Сменить мастер-пароль"
- Ввод текущего пароля (неправильный) → ошибка
- Ввод нового (короткий) → ошибка "минимум 12 символов"
- Ввод правильного + нового (12+) + подтверждение → успех
- Войти заново с новым паролем → работает

### 9. QR-коды
- Открыть запись → "Показать QR-код"
- QR **не содержит пароль** (проверить через обычную камеру)
- QR содержит только технический токен
- "Сканировать QR" в меню → камера открывается
- Навести на QR → запрос мастер-пароля
- После подтверждения → пароль виден
- Изменить пароль → старый QR становится недействительным

### 10. Импорт/Экспорт
-  Экспорт → предупреждение о безопасности
-  Удалить записи
-  Импорт → записи появляются в **выбранном профиле**
-  Пароли открываются (на том же устройстве)

### 11. Валидация паролей
-  Попробовать ввести пароль с повторами → ошибка
-  Попробовать ввести пароль, который уже был → ошибка
-  Попробовать ввести пароль, похожий на предыдущий (<60%) → ошибка
