# SecureVault — Менеджер паролей с AMPG v2

Безопасный менеджер паролей для Android с авторским алгоритмом мнемонической генерации.

Проект разработан в рамках магистерской диссертации.

**Репозиторий:** https://github.com/MAN2ll/SecureVault-Final

---

## Цель проекта

Создать безопасное Android-приложение для локального хранения паролей с двумя режимами генерации:

1. **Обычный генератор** — криптостойкие случайные пароли без повторяющихся символов.
2. **Мнемонический генератор AMPG v2** — запоминающиеся пароли по пользовательской фразе с гарантированным отсутствием повторов символов.

Приложение решает проблему баланса между безопасностью (сложные пароли) и удобством (возможность запомнить пароль по подсказке).

---

## Основные функции

### Генерация паролей
- **Обычный генератор**: SecureRandom, без повторяющихся символов (A и a считаются повтором).
- **Мнемонический генератор AMPG v2**: детерминированная генерация по фразе с уникальным потоком символов.
- **5 вариантов** в каждом наборе с разными стратегиями.
- **Кнопка "Ещё варианты"** для перегенерации с номером набора.

### Ротация паролей
- Индивидуальная ротация с напоминаниями (3, 6, 12 месяцев).
- Массовая ротация с предпросмотром.
- **Перемешивание паролей между сервисами** внутри профиля.
- Автоматическое обновление `nextRotationDate` иhistory.

### История паролей
- Зашифрованное хранение старых паролей (AES-256-GCM).
- Просмотр только после подтверждения мастер-пароля.
- Типы операций: `random`, `mnemonic`, `manual`, `shuffled`.

### Безопасность
- AES-256-GCM через Android Keystore для шифрования паролей.
- PBKDF2WithHmacSHA256 (100,000 итераций) для мастер-пароля.
- HMAC-SHA256 для fingerprint паролей (ключ в Android Keystore).
- Constant-time compare для защиты от timing attacks.
- BruteForceGuard: блокировка после неудачных попыток.

### QR-коды
- QR-код **НЕ содержит пароль** — только защищённый токен-указатель
- QR открывает **карточку записи** внутри SecureVault
- QR работает **только на этом устройстве** и в **этом профиле**
- Для просмотра пароля нужен **мастер-пароль**
- QR **остаётся действительным после ротации пароля** — можно распечатать один раз
- После сканирования показывается **актуальный пароль** записи
- QR можно сохранить как PNG-изображение через кнопку "Сохранить QR-код"
- QR привязан к устройству: после переноса на другое устройство QR нужно создать заново

### Импорт/Экспорт
- CSV с поддержкой метаданных (fingerprint, mnemonicPhraseHint).
- Генерация новых UUID при импорте.

### Аудит безопасности
- Поверхностный аудит по метаданным (без расшифровки).
- Глубокий аудит после подтверждения мастер-пароля (длина, повторы, структура).

## AMPG v2 — Unique Mnemonic Flow

**AMPG v2** (Adaptive Mnemonic Password Generation v2) — это авторский алгоритм мнемонической генерации.

> **Важно понимать:** AMPG — это генератор пароля, а НЕ криптография. Алгоритм предназначен для создания запоминаемых паролей. Для максимальной криптостойкости используйте обычный случайный генератор.

### Идея алгоритма
Алгоритм строит уникальный поток символов из фразы, затем применяет распределённые замены по всему паролю (начало, середина, конец). Повторяющиеся символы заменяются знаками/цифрами или пропускаются. Итоговый пароль гарантированно не содержит повторяющихся символов (проходит строгую проверку `PasswordValidator.hasDuplicateCharacters`).

### Этапы генерации

1. **Нормализация фразы:** удаление лишних пробелов, приведение к нижнему регистру.
2. **Транслитерация:** перевод кириллицы в латиницу, фильтрация небуквенных символов.
3. **Построение уникального потока:** проход по фразе слева направо, включение только тех букв, которых ещё не было (без учёта регистра).
4. **Распределённые замены:** примерно 30% символов заменяются по фиксированным правилам:
   - `a -> @`, `o -> 0`, `e -> 3`, `i -> 1`, `u -> ^`, `s -> $`, `y -> 7`
   - `t -> +`, `l -> !`, `b -> 8`, `g -> 9`, `z -> 2`
   - Замены детерминированно распределяются по началу, середине и концу пароля.
5. **Код сервиса:** добавление уникальных символов названия сервиса и его длины.
6. **Код ротации:** добавление даты (MMYY или QYY) с проверкой на повторы цифр.
7. **Финальная очистка (sanitize):** удаление оставшихся повторов и добивание длины свободными символами.

   
### Удаление записей
- Удаление одной записи — только после подтверждения мастер-паролем
- Массовое удаление (долгий тап или меню "Выбрать записи") — только после мастер-пароля
- Удаление всех паролей профиля — только после мастер-пароля
- Удаление профиля — только если в нём **нет паролей**, и только после мастер-пароля
- Все опасные действия требуют подтверждения мастер-паролем
  
### Пример работы

**Входные данные:**
- Фраза: "моя кошка любит рыбу"
- Сервис: Gmail

**Транслитерация и уникальный поток:**
`m o y a k s h l u b i t r`

**Итоговый пароль (пример):**
`M07@k$h!ub1tr-G5`

**Подсказка:**
`моя кошка любит рыбу + Gmail + правило уникального потока + распределённые замены`

### 5 стратегий вариантов
1. Уникальный поток + мягкие замены.
2. Уникальный поток + полные замены.
3. Код сервиса в начале.
4. Код ротации в начале.
5. Компактный вариант (только инициалы слов).

## Безопасность

### Мастер-пароль
- Минимум 12 символов.
- PBKDF2WithHmacSHA256 (100,000 итераций), случайная соль 16 байт.
- BruteForceGuard: прогрессивная блокировка (30 сек, 1 мин, сброс доступа).

### Шифрование и Fingerprint
- AES-256-GCM с уникальным IV для каждого пароля.
- HMAC-SHA256 для отпечатков (fingerprint) паролей.
- Проверка истории и уникальности работает по fingerprint, без расшифровки.

### Профильный PIN
- Короткий локальный PIN (4-8 символов) для разделения профилей.
- НЕ является заменой мастер-пароля. Мастер-пароль защищает всё хранилище.

### QR-коды
- QR не содержит пароль, только защищённый токен (ID записи, хеш профиля, nonce, метку времени).
- Действует до смены пароля. После ротации старый QR недействителен.

## Ограничения

1. **Импорт/Экспорт CSV:** не предназначен для безопасной передачи между устройствами, так как Android Keystore привязан к конкретному устройству.
2. **BruteForceGuard:** после 10 неудачных попыток происходит сброс доступа (очистка сессии), но сама база зашифрованных паролей не удаляется.
3. **Мнемонические пароли:** менее криптостойкие, чем полностью случайные. Предназначены для баланса удобства и безопасности.

## Сценарии ручной проверки

### 1. Генерация AMPG v2
- [ ] Открыть мнемонический генератор.
- [ ] Ввести фразу "моя кошка любит рыбу" и сервис "Gmail".
- [ ] Убедиться, что сгенерировано 5 вариантов.
- [ ] Проверить, что замены (цифры/символы) есть в начале, середине и конце пароля.
- [ ] Сохранить вариант.

### 2. Ротация и подсказки
- [ ] Открыть запись, созданную через AMPG v2.
- [ ] Запустить ротацию, выбрать "Мнемонический".
- [ ] Ввести новую фразу и выбрать новый вариант.
- [ ] Убедиться, что после замены пароль, текстовая подсказка и mnemonicPhraseHint обновились.

### 3. Изоляция профилей
- [ ] Создать Профиль 1 и Профиль 2.
- [ ] Добавить запись в Профиль 1.
- [ ] Выйти и войти в Профиль 2.
- [ ] Убедиться, что запись из Профиля 1 не отображается.

### 4. QR-коды
- [ ] В списке записей нажать меню (три точки) у нужного сервиса.
- [ ] Выбрать "QR-код".
- [ ] Убедиться, что отображается QR и текст "QR действует до смены пароля".
- [ ] Отсканировать QR через встроенный сканер, ввести мастер-пароль и получить доступ.
